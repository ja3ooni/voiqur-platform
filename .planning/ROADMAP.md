# Roadmap: EUVoice AI / Voiquyr Platform

## Overview

The platform is ~55% complete with a working skeleton (billing, analytics, compliance structure, frontend shell) but all core voice AI runs on mock data. This roadmap drives the remaining 45% — replacing every mock with production implementations across 10 phases: foundation plumbing, voice pipeline (STT → LLM → TTS), telephony, integrations, frontend, tests, and production hardening. End state: a deployable, EU-resident, GDPR-compliant voice AI platform ready for MedGulf, Oman LNG, and Aqaba client deployments.

### Phase 1: Foundation
**Goal**: The platform boots against real infrastructure with valid secrets, live DB connections, and a working auth flow
**Depends on**: Nothing (first phase)
**Requirements**: FOUND-01, FOUND-02, FOUND-03, FOUND-04, FOUND-05, FOUND-06, FOUND-07, FOUND-08, FOUND-09
**Success Criteria** (what must be TRUE):
  1. `python -m uvicorn src.api.main:app --reload` starts without errors when `.env` contains all required vars
  2. `/health` endpoint returns `{"redis": "ok", "postgres": "ok"}` against live Redis and PostgreSQL instances
  3. `POST /auth/register` stores a bcrypt-hashed user row in the `users` table
  4. `POST /auth/login` returns a signed JWT that `/auth/me` (or any protected route) accepts
  5. All required environment variables are documented in `.env.example` with descriptions
